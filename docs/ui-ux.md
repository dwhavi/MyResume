# UI/UX — MyResume

## 스텝퍼 구조

4단계 워크플로우, URL searchParams로 step 관리:

```
Step 1: 채용공고 입력     [①②③④]
Step 2: 프로필 입력       [①②③④]
Step 3: 공고 분석 결과    [①②③④]
Step 4: 이력서 미리보기    [①②③④]
```

### Step 1 — 채용공고 입력

- 대형 textarea (placeholder에 예시 텍스트)
- 글자 수 카운터 (현재/10,000)
- 10,000자 초과 시 에러 표시
- "다음" 버튼

### Step 2 — 프로필 입력

섹션별로 구분된 폼:

**기본정보**
- 이름 (input, autocomplete="name")
- 연락처 (input type="tel", autocomplete="tel")
- 이메일 (input type="email", autocomplete="email")
- 생년월일 (input type="date" 또는 YYYY-MM-DD 텍스트)
- 주소 (input, autocomplete="street-address")

**사진 업로드**
- 드래그앤드롭 + 클릭 업로드
- react-easy-crop으로 크롭 (3:4 비율)
- 모바일: 카메라/갤러리 선택 (`accept="image/*"`, `capture` 속성)
- 미리보기 표시
- 삭제 버튼

**자기소개**
- textarea (500자 이내, 글자 수 카운터)
- placeholder: "지원 동기와 핵심 역량을 간단히 작성해주세요 (생략 가능)"

**기술 스택**
- 동적 추가/삭제
- 각 항목: 기술명 (input) + 숙련도 (select: 상/중/하) + 경력년수 (input number)

**경력**
- 동적 추가/삭제
- 각 항목: 회사명, 직무, 기간(시작~종료), 경력기술 (bullets, 동적 추가)
- 종료일 미입력 시 "재직중" 처리

**학력**
- 동적 추가/삭제
- 각 항목: 학교명, 학위, 전공, 졸업년도

**프로젝트**
- 동적 추가/삭제
- 각 항목: 프로젝트명, 역할, 기간, 설명, 기술스택

**자격증**
- 콤마로 구분된 텍스트 입력

### Step 3 — 공고 분석 결과

AI 분석 완료 후 표시:

```
📋 공고 분석 결과

필수 기술:    [TypeScript] [React] [Node.js]
우대 기술:    [AWS] [Docker] [CI/CD]
핵심 키워드:  [풀스택] [REST API] [Agile]

주요 업무:
• 백엔드 API 설계 및 개발
• 프론트엔드 React 컴포넌트 개발
• ...

자격 요건:
• 3년 이상 웹 개발 경험
• TypeScript 숙련
• ...
```

- "이력서 생성하기" 버튼

### Step 4 — 이력서 미리보기 + PDF

**정합성 점수**
```
채용공고와의 정합성: 87% 🎯

✅ TypeScript, React, CI/CD 경험 언급됨
⚠️ '팀 리딩 경험'이 공고에 있지만 구체적 사례 없음
💡 '성능 최적화 경험'을 추가하면 매칭도 상승 예상
```

**미리보기**
- A4 비율 컨테이너에 렌더링
- 섹션별 인라인 편집 가능 (contentEditable)
- 2장 초과 시 경고

**액션 버튼**
- PDF 다운로드
- 새 탭에서 열기
- "이력서 다시 생성" (전체 재생성)

## 로딩 UX

| 단계 | 로딩 메시지 |
|------|-----------|
| 공고 분석 중 | "채용공고를 분석하고 있어요..." |
| 이력서 생성 중 | "맞춤 이력서를 작성하고 있어요..." (15~30초) |
| PDF 변환 중 | "PDF를 생성하고 있어요..." |

구현: Skeleton UI + 단계별 메시지 (setTimeout으로 순차 표시)

## 에러 UX

| 상황 | 표시 |
|------|------|
| API 실패 (재시도 후) | "이력서 생성에 실패했습니다. 다시 시도해주세요." + 재시도 버튼 |
| 타임아웃 | "생성 시간이 초과되었습니다. 공고를 짧게 줄이거나 다시 시도해주세요." |
| Rate limit | "요청이 너무 많습니다. 1분 뒤 다시 시도해주세요." |
| 네트워크 오류 | "인터넷 연결을 확인해주세요." |

**그레이스풀 폴백:** AI 생성 실패 시 사용자가 입력한 프로필을 기반으로 한 기본 초안 템플릿을 보여주고 수동 편집 유도.

## 반응형

| 브레이크포인트 | 동작 |
|--------------|------|
| Desktop (≥1024px) | 미리보기 좌/우 분할 |
| Tablet (768~1023px) | 세로 스택 |
| Mobile (<768px) | 전체 화면 폼, 하단 고정 "다음" 버튼 |

## 모바일 전략

- inputmode: `tel`, `email`, `numeric` 적용
- textarea vs input 구분 (긴 텍스트는 textarea)
- 사진: `accept="image/*"` + `capture` 속성
- 빈 폼에 가이드 텍스트 표시 (이탈 방지)
- sessionStorage 한계 알림: "다운로드 전에 PDF를 저장해주세요. 탭을 닫으면 데이터가 초기화됩니다."

## 색상/디자인 (Tailwind)

- 기본: 다크 그레이 + 블루 액센트
- 폰트: 시스템 한글 폰트 (sans)
- 카드 스타일: rounded-lg + shadow-sm
- 버튼: primary (blue), secondary (gray)
