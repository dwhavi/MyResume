# Development Log — 2026-04-09

오늘 진행된 MyResume 프로젝트의 핵심 기능 추가 및 AI 파이프라인 안정화 작업 내역을 집대성합니다.

---

## 🚀 주요 마일스톤

### 1. 프로필 분석 파이프라인 구축 (Profile Intelligence)
- **PDF 분석**: `pdf-parse`를 활용하여 기존 이력성/경력기술서/포트폴리오에서 텍스트를 추출하고 AI로 구조화된 프로필 데이터를 생성하는 기능 추가.
- **URL 스크래핑**: Playwright를 사용하여 공고 페이지의 텍스트와 이미지를 캡처, 핵심 직무 요구사항을 분석하는 기능 추가.
- **Markdown 복원**: 내보낸 Markdown 파일을 다시 읽어와 프로필을 수정/보완할 수 있는 데이터 영속성 강화.

### 2. AI 파이프라인 안정화 (Reliability)
- **에러 핸들링**: 404(모델명 오류) 해결을 위해 `gemini-flash-latest`로 식별자 통일.
- **과부하 대응**: 503(Service Unavailable) 발생 시 자동 재시도 로직 및 사용자 친화적 한국어 안내 메시지 적용 (GR-11).
- **아키텍처 준수**: 모든 프롬프트를 `src/server/prompts.ts`로 중앙 집중화하여 관리 효율성 증대 (GR-8).

### 3. PDF 생성 엔진 고도화 (Export Quality)
- **컬러 호환성 해결**: `html2canvas`가 최신 CSS `lab()`, `oklch()` 색상을 처리하지 못해 발생하는 오류 해결.
- **가시성 보정**: 비표준 색상을 실시간으로 감지하여 표준 가시 색상(#333 등)으로 폴백하는 스타일 가드 로직 적용.

## 🐞 해결된 주요 장애 (Incident History)

| 장애 현상 | 원인 분석 | 조치 사항 |
| :--- | :--- | :--- |
| **PDF Parse ENOENT** | `pdf-parse` v1.1.1 로드 시 내부 테스트 코드 오작동 | 모듈 엔트리 우회 (`lib/pdf-parse.js` 직접 참조) |

| **Model 404 Error** | `gemini-1.5-flash` 식별자 인식 불안정 | `gemini-flash-latest` 고정 사용 및 재시도 로직 보강 |
| **Prompt Missing** | 리팩토링 중 `SYSTEM_PROMPT` 누락 | 설계 문서 기반 복구 및 통합 프롬프트 관리 체계 구축 |
