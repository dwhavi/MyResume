# Plans — MyResume

실행 계획은 **1급 아티팩트**입니다. 에이전트가 작업 전에 계획을 세우고, 진행 상황과 의사결정을 기록합니다.

## 계획 템플릿

```markdown
# [계획명]

## 상태
- **상태:** in-progress | done | blocked
- **생성일:** YYYY-MM-DD
- **완료일:** YYYY-MM-DD (완료 시)

## 목표
[무엇을 달성할 것인가]

## 작업 항목
- [ ] 항목 1
- [ ] 항목 2
- [x] 완료된 항목

## 의사결정 로그
### YYYY-MM-DD — [결정 내용]
- 맥락: 왜 이 결정을 내렸는지
- 대안: 고려했던 다른 선택지
- 결과: 무엇을 선택했는지
```

## 계획 규칙

1. **간단한 변경** (버그 수정, 스타일 조정 등)은 계획 없이 진행 가능
2. **새 기능, 아키텍처 변경, 여러 파일에 걸친 작업**은 계획 필수
3. 계획 파일명: `docs/plans/YYYY-MM-DD-[간단한설명].md`
4. 완료 후 `done`으로 변경, 필요시 `docs/decisions.md`에 핵심 결정 반영
5. 블록된 계획은 `blocked` + 블록 사유 기록

## 진행 중 / 완료된 계획

> 코딩 에이전트가 작업을 시작하면 여기에 계획이 생성됩니다.
