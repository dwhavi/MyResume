# Skill: 한국어 IT 이력서 포맷

## 항목 순서 (절대 변경 금지)

```
1. 기본정보 (이름, 사진, 연락처, 이메일)
2. 자기소개 / 지원동기
3. 학력
4. 경력 (최신순)
5. 기술 스택
6. 프로젝트
7. 자격증
```

## 자기소개 작성 원칙

- 500자 이내
- 공고에 맞춰 매번 다르게 작성
- 구조: 지원동기 → 핵심 역량 → 입사 후 포부
- **금지:** "저는 ~한 사람입니다"로 시작하는 진부한 문장
- **금지:** 과도한 수식어 ("완벽한", "최고의")
- **권장:** 구체적 숫자/성과 포함 ("3년간 ~개의 서비스 운영")

## 경력 기술서 작법

- 각 항목 200자 이내
- STAR 기법 권장 (상황→과제→행동→결과)
- 구체적 수치 포함 ("응답 시간 40% 개선", "DAU 5만명 달성")
- 본인이 한 것만 작성 (팀 성과 본인 기여도 구분)

## 기술 스택 표기

- 카테고리별 그룹화 (언어/프레임워크/DB/인프라/툴)
- 숙련도 명시: 상/중/하 또는 ⭐⭐⭐ / ⭐⭐ / ⭐
- 최신 사용 기술 우선

## 프로젝트 설명

- 프로젝트명, 본인 역할, 기간, 사용 기술
- 2~3줄로 핵심만
- 본인 기여 명확히

## 톤 가이드

| 타겟 | 톤 | 문체 |
|------|-----|------|
| 대기업 | 격식 있게 | ~습니다/~입니다 |
| 스타트업 | 친근하게 | ~해요/~입니다 |
| 글로벌 | 영문혼합 | 기술명 영문 유지 |

MVP에서는 기본적으로 **~습니다/~입니다** (격식체) 사용.

## A4 레이아웃 규칙

- 총 1~2장
- 여백: 상하좌우 15mm
- 본문 폰트: 10~11pt
- 제목: 14~16pt, 볼드
- 섹션 구분선: 얇은 회색 라인
- 사진: 3×4cm, 우상단 또는 좌상단
- 행간: 1.5~1.6

## 절대 하지 말 것

- 영문 이력서 순서 적용 (Education → Experience → Skills 순은 한국에서 어색함)
- Objective/Summary를 상단에 거대하게 배치
- 불필요한 아이콘/이미지 과다 사용
- 칼라 과다 사용 (흑백+포인트 컬러 1개 권장)
