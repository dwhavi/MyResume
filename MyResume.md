# 📄 AI 이력서 맞춤 생성기 — 프로젝트 기획서 v2

> 채용공고를 분석하고, 사용자 경력/스킬에 맞춰 AI가 맞춤 이력서를 생성하는 웹 앱
> 
> Generated by 3-agent discussion: 혁신가 🔴 · 실용주의자 🟢 · main 📋

---

## 1. 프로젝트 개요

**핵심 가치:** "채용공고 하나 → 맞춤 이력서 하나"

**1차 타겟 사용자:** IT/개발 직무 구직자 (주니어~미드 레벨)
- 경력 1~7년의 프론트엔드/백엔드/풀스택 개발자
- 채용공고마다 이력서를 맞춤 수정해야 하는 파이프라인 문제 겪는 중

**성공 지표 (KPI):**
- 첫 주 100명 이상 사용
- 평균 이력서 생성 시간 < 10초
- 완성률 (PDF 다운로드까지 도달) > 60%
- 재방문률 > 30%

- 사용자가 경력/스킬을 입력하고, 채용공고 텍스트를 붙여넣기
- AI가 공고 키워드를 분석 → 정합성 점수 도출 → 최적화된 이력서 자동 생성
- 미리보기 확인 후 PDF 다운로드

## 2. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| 프론트엔드 | Next.js 14+ (App Router) + TypeScript + Tailwind CSS | 풀스택 단일 프로젝트, 반응형 |
| 상태 관리 | useState + sessionStorage | MVP에 DB 없음, 새로고침 유지 |
| AI | OpenAI API (gpt-4o-mini) | 비용 효율, 한국어 충분 |
| PDF | html2canvas + jspdf | 한글 폰트 안정, React-PDF보다 신뢰성 높음 |
| 배포 | Vercel (Hobby Plan) | 무료, 자동 배포 |

## 3. 사용자 시나리오

```
① 채용공고 텍스트 붙여넣기
② AI 공고 분석 → 키워드/요구사항 추출 + 정합성 점수 표시
③ 사용자가 분석 결과 확인/수정
④ 프로필 입력 (구조화된 폼, 사진 업로드 포함)
⑤ AI 이력서 자동 생성 (로딩 UI + 진행 메시지)
⑥ 미리보기 확인 + 부분/전체 수정 가능
⑦ PDF 다운로드 (A4 1~2장)
```

## 5. API 설계

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/resumes/generate` | 공고+프로필 → 공고 분석 + 이력서 JSON 생성 (일원화) |
| POST | `/api/resumes/regenerate` | 특정 섹션만 재생성 (V2) |
| GET | `/api/resumes/:id/pdf` | PDF 변환/다운로드 |

### 핵심 데이터 흐름

```
POST /api/resumes/generate
Request: { jobDescription, profile, regenerateSection? }

→ 1단계: 공고 분석 (내부)
   - 키워드, 요구사항, 우대사항 추출
   - 프로필과 매칭하여 정합성 점수 계산

→ 2단계: 이력서 생성
   - OpenAI API 호출 (system prompt + few-shot)
   - response_format: { type: "json_object" }
   - temperature: 0.3~0.5

→ 3단계: 응답 검증
   - Zod 스키마 검증
   - 실패 시 1회 재시도

Response: { analysis, resume, matchScore }
```

## 6. 프롬프트 설계

### 시스템 프롬프트

```
당신은 10년 경력의 IT 채용 컨설턴트입니다.
삼성, 네이버, 카카오 등 국내 대기업과 유니콘 스타트업의
이력서를 500건 이상 검토한 전문가입니다.

[출력 규칙]
- 반드시 JSON 형식으로 출력하세요
- 한국어 이력서 표준 순서: 기본정보 → 자기소개 → 학력 → 경력 → 기술스택 → 프로젝트 → 자격증
- 총 A4 2장 이내로 작성
- 자기소개 500자 이내, 경력 기술 200자/항목 이내

[절대 하지 말 것]
- 사용자가 제공하지 않은 정보를 만들어내지 마세요
- 과도한 자기PR 문구를 사용하지 마세요
- 공고에 없는 기술을 이력서에 포함하지 마세요
```

### 프롬프트 튜닝 포인트
- **Few-shot 예시 2~3개** 포함 (실제 좋은 한국어 이력서 예시)
- **temperature 0.3~0.5** (창의성보다 일관성)
- **체인 분리**: 공고 분석 프롬프트와 이력서 생성 프롬프트를 내부적으로 2단계로 분리
- **부정적 지시사항** 명시 (hallucination 방지)

## 7. 데이터 모델

```
Profile (입력 데이터)
├── name, phone, email, birthdate, address, gender

├── education: [{school, degree, field, year}]
└── projects: [{name, role, description, techStack[]}]

JobAnalysis (AI 공고 분석 결과)
├── required_skills[]
├── preferred_skills[]
├── keywords[]
├── responsibilities[]
└── qualifications[]

Resume (생성 결과)
├── content: JSON (구조화된 이력서)
├── match_score: number (0~100, 정합성 점수)
├── matched_keywords[] (매칭된 키워드)
├── missing_keywords[] (누락된 키워드)
└── suggestions[] (개선 제안)
```

## 9. 비기능 요구사항

| 관점 | 요구사항 |
|------|----------|
| **성능** | 이력서 생성 < 15초, 타임아웃 60초 |
| **보안** | API 키 서버사이드만, Rate limiting (분당 3회) |
| **프롬프트 방어** | 채용공고 텍스트에 프롬프트 주입 방어 (입력 sanitize + 시스템/유저 메시지 분리 + 특수문자 필터링), 입력 길이 제한 10000자 |
| **사용성** | 반응형, 한국어, 모바일 기본 지원, autocomplete 속성 적용 |
| **PDF** | A4, 한글 폰트 안정, 새 탭에서 열기 |
| **에러 처리** | API 실패 시 1회 재시도 → 실패 시 기본 템플릿 폴백, JSON 파싱 실패 시 1회 재시도, 사용자에게 실패 사유 명시 |

### API 에러 코드 체계

| 에러 케이스 | HTTP | 사용자 메시지 |
|-------------|------|-------------|
| API 키 만료/오류 | 500 | "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |
| 입력 길이 초과 | 400 | "채용공고는 10,000자 이내로 입력해주세요." |
| JSON 파싱 실패 | 502 | "이력서 생성에 실패했습니다. 다시 시도해주세요." |
| Rate limit 초과 | 429 | "요청이 너무 많습니다. 1분 뒤 다시 시도해주세요." |
| 타임아웃 (60초) | 504 | "생성 시간이 초과되었습니다. 공고를 짧게 줄이거나 다시 시도해주세요." |

### 사진 처리 명세

| 항목 | 스펙 |
|------|------|
| 비율 | 3:4 (세로) |
| 권장 해상도 | 300×400px 이상 |
| 파일 형식 | JPG, PNG |
| 최대 용량 | 5MB |
| 압축 | 업로드 시 클라이언트에서 Canvas API로 리사이즈 + 압축 |
| 크롭 라이브러리 | react-easy-crop |
| 기본 자르기 | center-crop, 정중앙 기준 |

### 모바일 폼 전략

- 입력 필드: textarea (경력/프로젝트) vs input (이름/연락처) 구분
- autocomplete 속성: `name`, `tel`, `email`, `organization-title` 등 적용
- 사진 업로드: 카메라/갤러리 선택 지원 (`accept="image/*"`, `capture` 속성)
- 키보드 타입: `type="tel"`, `type="email"`, `inputmode="numeric"` 적용
- 긴 폼은 섹션별로 분리 (스텝퍼 구조 활용)

| ChatGPT 직접 입력 | 프롬프트 수동 | 구조화된 폼 + 정합성 점수 + PDF 출력 자동화 |

## 17. 토론 결정 사항

### 🔴 혁신가
- "정합성 점수가 핵심 차별화. 이거 없으면 그냥 ChatGPT 붙여넣기와 같음"

- "1주에 5개 기능만. 그중 3개를 완벽하게"
- "gpt-4o-mini + temperature 0.3~0.5"
- "프롬프트 주입 방어 필수"

- "GAP 분석을 선택이 아니라 핵심으로 승격"
